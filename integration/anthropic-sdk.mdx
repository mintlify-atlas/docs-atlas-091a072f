While the Anthropic SDK doesn't natively support custom base URLs, you can use vLLora's OpenAI-compatible API to trace and monitor Claude models. This guide shows two approaches: using the OpenAI SDK format or using HTTP requests directly.

## Approach 1: OpenAI SDK Format (Recommended)

The easiest way to use Claude through vLLora is with the OpenAI SDK format:

```python
from openai import OpenAI

# Use OpenAI SDK pointed at vLLora with Claude model
client = OpenAI(
    api_key="your-anthropic-api-key",  # Configure in vLLora UI
    base_url="http://localhost:9090/v1"
)

response = client.chat.completions.create(
    model="claude-opus-4-5-20251101",
    messages=[
        {"role": "user", "content": "Hello, Claude!"}
    ]
)

print(response.choices[0].message.content)
```

## Approach 2: Direct HTTP Requests

Use Python's `requests` library to make direct calls:

```python
import requests
import json

url = "http://localhost:9090/v1/chat/completions"

payload = {
    "model": "claude-opus-4-5-20251101",
    "messages": [
        {"role": "user", "content": "What is the capital of France?"}
    ]
}

headers = {
    "Content-Type": "application/json"
}

response = requests.post(url, json=payload, headers=headers)
data = response.json()

print(data["choices"][0]["message"]["content"])
```

## Streaming Responses

vLLora supports streaming with Claude models:

```python
from openai import OpenAI

client = OpenAI(
    api_key="your-anthropic-api-key",
    base_url="http://localhost:9090/v1"
)

stream = client.chat.completions.create(
    model="claude-opus-4-5-20251101",
    messages=[{"role": "user", "content": "Write a short story"}],
    stream=True
)

for chunk in stream:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="", flush=True)
```

## Tool Use (Function Calling)

vLLora translates OpenAI-style function calls to Claude's tool use format:

```python
from openai import OpenAI

client = OpenAI(
    api_key="your-anthropic-api-key",
    base_url="http://localhost:9090/v1"
)

tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "Get the current weather in a location",
            "parameters": {
                "type": "object",
                "properties": {
                    "location": {
                        "type": "string",
                        "description": "City name"
                    },
                    "unit": {
                        "type": "string",
                        "enum": ["celsius", "fahrenheit"]
                    }
                },
                "required": ["location"]
            }
        }
    }
]

response = client.chat.completions.create(
    model="claude-opus-4-5-20251101",
    messages=[{"role": "user", "content": "What's the weather in Paris?"}],
    tools=tools
)

if response.choices[0].message.tool_calls:
    for tool_call in response.choices[0].message.tool_calls:
        print(f"Tool: {tool_call.function.name}")
        print(f"Arguments: {tool_call.function.arguments}")
```


### Multi-turn Conversations

```python
from openai import OpenAI

client = OpenAI(
    api_key="your-anthropic-api-key",
    base_url="http://localhost:9090/v1"
)

messages = [
    {"role": "user", "content": "What is Python?"},
]

# First turn
response1 = client.chat.completions.create(
    model="claude-opus-4-5-20251101",
    messages=messages
)

messages.append({
    "role": "assistant",
    "content": response1.choices[0].message.content
})
messages.append({
    "role": "user",
    "content": "What are its main features?"
})

# Second turn
response2 = client.chat.completions.create(
    model="claude-opus-4-5-20251101",
    messages=messages
)

print(response2.choices[0].message.content)
```

## Async Support

Use async clients for concurrent requests:

```python
import asyncio
from openai import AsyncOpenAI

async def ask_claude(question: str) -> str:
    client = AsyncOpenAI(
        api_key="your-anthropic-api-key",
        base_url="http://localhost:9090/v1"
    )
    
    response = await client.chat.completions.create(
        model="claude-opus-4-5-20251101",
        messages=[{"role": "user", "content": question}]
    )
    
    return response.choices[0].message.content

async def main():
    questions = [
        "What is AI?",
        "Explain machine learning",
        "What is deep learning?"
    ]
    
    tasks = [ask_claude(q) for q in questions]
    answers = await asyncio.gather(*tasks)
    
    for q, a in zip(questions, answers):
        print(f"Q: {q}")
        print(f"A: {a}\n")

asyncio.run(main())
```

## Using with LangChain

Integrate Claude through vLLora with LangChain:

```python
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(
    model="claude-opus-4-5-20251101",
    openai_api_key="your-anthropic-api-key",
    base_url="http://localhost:9090/v1"
)

response = llm.invoke("What is the meaning of life?")
print(response.content)
```

## Supported Parameters

vLLora supports the following Claude-specific parameters:

| Parameter | Supported | Notes |
|-----------|-----------|-------|
| `model` | ✓ | All Claude models |
| `messages` | ✓ | Standard format |
| `max_tokens` | ✓ | Output length limit |
| `temperature` | ✓ | 0.0 to 1.0 |
| `top_p` | ✓ | Nucleus sampling |
| `top_k` | ✓ | Via provider_specific |
| `stop_sequences` | ✓ | Stop generation |
| `stream` | ✓ | Streaming responses |
| `tools` | ✓ | Function calling |

## Troubleshooting

### API Key Configuration

Ensure your Anthropic API key is set in vLLora UI at `http://localhost:9091`.

### Model Not Found

Verify the model name matches Anthropic's model identifiers. Check available models in the vLLora UI.

### Connection Issues

Test vLLora connectivity:

```bash
curl http://localhost:9090/health
```

### Rate Limits

vLLora respects Anthropic's rate limits. Check the vLLora UI for rate limit information and consider implementing retry logic.
