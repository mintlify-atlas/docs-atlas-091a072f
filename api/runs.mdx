title: Runs

Runs represent complete execution cycles of your AI workflows, aggregating usage metrics, costs, and performance data across all operations.

## List Root Runs

Retrieve root runs (top-level executions) with optional filtering.

<CodeGroup>
```bash cURL
curl -X GET "http://localhost:3002/runs?limit=50&offset=0" \
  -H "x-project-slug: default"
```

```python Python
import requests

headers = {"x-project-slug": "default"}
params = {
    "limit": 50,
    "offset": 0,
    "modelName": "gpt-4",
    "typeFilter": "llm"
}

response = requests.get(
    "http://localhost:3002/runs",
    headers=headers,
    params=params
)
data = response.json()
```
</CodeGroup>

## Get Run Details

Retrieve detailed information about a specific run.

<CodeGroup>
```bash cURL
curl -X GET "http://localhost:3002/runs/{run_id}/details" \
  -H "x-project-slug: default"
```

```python Python
import requests

run_id = "run_abc123"
headers = {"x-project-slug": "default"}

response = requests.get(
    f"http://localhost:3002/runs/{run_id}/details",
    headers=headers
)
run_details = response.json()
```
</CodeGroup>

<ParamField path="run_id" type="string" required>
  The run ID to retrieve
</ParamField>

<ResponseField name="run" type="object">
  The run object (see List Root Runs for structure)
</ResponseField>

## Usage Metrics

Runs provide comprehensive usage metrics:

### Token Usage
- **input_tokens** - Total tokens sent to LLM APIs
- **output_tokens** - Total tokens received from LLM APIs

### Cost Tracking
- **cost** - Total cost calculated based on model pricing
- Includes both input and output token costs
- Supports custom model pricing via project settings

### Performance Metrics
- **llm_calls** - Number of LLM API invocations
- **start_time_us** / **finish_time_us** - Execution duration
- **errors** - Any errors encountered during execution

## Filtering Examples

### Filter by Model

```bash
curl -X GET "http://localhost:3002/runs?modelName=gpt-4o-mini&limit=20" \
  -H "x-project-slug: default"
```

### Filter by Time Range and Labels

```bash
curl -X GET "http://localhost:3002/runs?startTimeMin=1709481600000000&labels=production" \
  -H "x-project-slug: default"
```

### Get Runs for Specific Threads

```bash
curl -X GET "http://localhost:3002/runs?threadIds=thread_1,thread_2" \
  -H "x-project-slug: default"
```

## Run Lifecycle

1. **Creation** - Run starts when a root span with a run_id is created
2. **Execution** - Spans are added as operations execute (LLM calls, tool executions, etc.)
3. **Completion** - Run finishes when all spans complete
4. **Aggregation** - Metrics are aggregated across all spans in the run

Runs automatically track:
- All models used
- All tools called
- Total token consumption
- Cumulative costs
- Error states
